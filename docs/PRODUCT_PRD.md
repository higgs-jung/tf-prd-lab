# PRODUCT PRD — Concept 6: “Geek Lab” (AI-Extensible Micro‑Experiments)

## 한 줄 요약
Next.js(Vercel)로 배포되는 **“긱스러운 실험(Experiment) 모음 웹”** 을 만들고, AI 팀(Planner→Captain→Worker→Judge)이 PR 단위로 **실험을 계속 추가/개선**한다.

## 제품 목표
- 방문자가 “오늘의 실험”을 눌러 **바로 만져볼 수 있는 데모**를 본다.
- 실험은 **작고 독립적인 모듈**(페이지/컴포넌트/MDX)로 추가되어, AI가 상상력을 발휘해도 구조가 무너지지 않는다.
- 모든 변경은 **PR 기반**으로 누적되고, 30분 루프에서 **결과만 알림**으로 전달된다.

## 타깃 사용자
- 개발자/메이커(“재미있는 시각화/툴/장난감”을 좋아하는 사람)
- 팀 오너(= 사용자): 결과만 보고 승인/중단 정도만 개입하고 싶은 사람

## 핵심 컨셉(“실험” 정의)
실험(Experiment)은 다음을 포함한다.
- `spec.md(x)`: 실험 목적/방법/입력/출력/제약/알고리즘(짧게)
- `Demo` 컴포넌트: 브라우저에서 동작하는 미니 앱(시각화/생성/조작)
- `meta`: 제목/태그/난이도/의존성(없으면 더 좋음)

“실험”은 **서로 최대한 독립**적이어야 한다(의존성 공유는 공용 UI 컴포넌트 정도로 제한).

## MVP 범위 (V1)
### In-scope
- Next.js(App Router) 기반 웹
- 실험 목록(카드) + 상세 페이지
- 태그/검색(클라이언트 사이드로 시작)
- “랜덤 실험” / “오늘의 실험” 진입
- 모든 실험은 레포 내 파일로 관리(초기에는 DB 없이)
- Vercel 배포(Preview/Production)

### Out-of-scope (V1)
- 회원가입/로그인, 사용자 데이터 영속 저장
- 외부 API 키/시크릿이 필요한 기능(유료 LLM 호출 등)
- 실험을 브라우저에서 자동 생성(생성은 PR로만 들어오게)

## 정보 구조(IA)
- `/` 홈: 오늘의 실험(1개) + 최신 추가(3개) + 태그
- `/experiments` 목록: 필터/검색/정렬
- `/experiments/[slug]` 상세: 스펙(문서) + 데모 + “관련 실험”
- `/about` 운영 원칙(“PR로만 추가된다”, “실험은 작게”)
- `/changelog` (옵션): PR 머지 기록을 사람이 읽기 좋게 요약

## 사용자 스토리
1. 방문자는 홈에서 “오늘의 실험”을 눌러 3초 내로 인터랙션 가능한 데모를 본다.
2. 방문자는 태그로 “viz / tools / weird / math” 같은 실험을 찾아본다.
3. 오너는 텔레그램 알림에서 “이번 30분에 머지된 실험/대기 PR/실패 PR”만 확인한다.
4. AI 팀은 “실험 1개 추가” 같은 작은 티켓을 PR로 처리해 누적한다.

## 성공 지표(초기)
- 첫 7일: 실험 10개 이상, 평균 실험 페이지 체류 30초 이상(대략)
- PR 루프: 하루 2~6개 PR 생성, 머지 실패율 20% 이하(초기 안정화 목표)
- 배포: Vercel Preview가 PR마다 생성되고, 최소 1개 smoke 체크가 돈다

## 품질/가드레일
- 실험은 **1 PR = 1 실험(또는 1 개선)** 원칙
- 번들 크기 폭증 방지: 실험 데모는 가능한 동적 import / 코드 스플리팅
- 접근성: 키보드 조작 가능한 기본 UI(슬라이더/버튼/입력)
- 성능: LCP 2.5s 내(일단 목표), 데모 로딩 스켈레톤 제공
- 보안: 사용자 입력을 DOM에 넣을 때 XSS 방지(예: dangerouslySetInnerHTML 금지)

## 기술 제안(권장)
- Next.js + TypeScript + App Router
- 스타일: Tailwind(또는 기존 선호가 있으면 교체)
- 실험 스펙: MDX(가능하면) 또는 Markdown + 프론트매터
- 실험 등록: `experiments/index.ts` 자동 수집(빌드 타임) 또는 파일 시스템 기반
- 최소 테스트: `pnpm test` 대신 “smoke + lint”부터(가볍게)

## 릴리즈 단계(예시)
### Milestone 0 — Repo Scaffolding
- Next.js 앱 생성, Vercel 배포 성공(Hello)
- PR마다 Preview 생성 + 간단한 Actions 체크 1개

### Milestone 1 — Experiments V1
- 실험 3개(각각 다른 유형: viz/tool/weird)
- 목록/상세/랜덤 진입 완성

### Milestone 2 — Curation Loop
- “오늘의 실험” 선정 규칙(최근/랜덤/태그 기반)
- 실험 메타 규칙(슬러그 충돌 방지, 태그 표준화)

## 티켓 템플릿(Planner가 쓰기 좋게)
각 티켓은 아래를 포함한다.
- 목표(1문장)
- 구현 범위(파일 단위)
- Done 정의(체크리스트)
- 검증 방법(로컬 커맨드/스크린샷)

## 자동화와의 연결(중요)
- Worker는 **브랜치+Draft PR** 로 실험을 추가한다.
- Judge는 체크리스트 기반 PASS/CHANGES REQUIRED를 낸다.
- Planner는 PASS면 **자동 머지**(필요 시 “체크 무시” 정책을 명시)한다.
